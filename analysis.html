üîÑ Data Flow: Accessing Process Automation Logs

üí° Important: How PA Stores Data

Process Automation stores its workflow logs in its own backend service, not in a database you can directly access. You must use PA APIs to retrieve workflow data (instances, tasks, decisions, etc.).

Three Approaches for Your Approval/Reject UI:

Approach 1: Direct PA API Access (Recommended) ‚≠ê

Step 1: User opens Approval UI (your custom SAPUI5 app)
‚Üì
Step 2: UI calls PA REST APIs
  • GET /v1/workflow-instances - Get all workflows
  • GET /v1/task-instances - Get pending tasks
  • GET /v1/workflow-instances/{id}/context - Get workflow data
‚Üì
Step 3: For each task, fetch AI reasoning from your custom database
  • GET /ai-service/reasoning/{workflowId}
‚Üì
Step 4: Display combined data (PA workflow + AI reasoning) in UI
‚Üì
Step 5: User approves/rejects via UI
‚Üì
Step 6: UI calls PA API to update task
  • PATCH /v1/task-instances/{id} with decision
Pros: Real-time data, no duplication, native PA integration
Cons: Limited complex queries, API rate limits

Approach 2: Hybrid (PA APIs + Your Database) ‚≠ê‚≠ê

Setup: Configure PA webhooks to notify your service on key events
‚Üì

üìä Database Schema for Custom Database

Regardless of which approach you choose, you'll need these tables in your custom database:

-- AI Reasoning logs (ALWAYS REQUIRED)
CREATE TABLE AI_REASONING_LOG (
    id UUID PRIMARY KEY,
    workflow_instance_id VARCHAR(100),  -- Link to PA workflow
    created_at TIMESTAMP,
    ai_model VARCHAR(50),
    input_data TEXT,
    reasoning_output TEXT,
    confidence_score DECIMAL,
    decision_recommendation VARCHAR(50)
);

-- Approval history (OPTIONAL - only for Hybrid approach)
CREATE TABLE APPROVAL_HISTORY (
    id UUID PRIMARY KEY,
    workflow_instance_id VARCHAR(100),
    task_id VARCHAR(100),
    ai_reasoning_id UUID,              -- Link to AI reasoning
    approver_user VARCHAR(100),
    decision VARCHAR(20),               -- 'APPROVED' | 'REJECTED'
    decision_timestamp TIMESTAMP,
    comment TEXT,
    pa_context TEXT                    -- Full PA context (JSON)
);

-- AI Chat history (ALWAYS REQUIRED if chat feature enabled)
CREATE TABLE AI_CHAT_LOG (
    id UUID PRIMARY KEY,
    workflow_instance_id VARCHAR(100),
    user_id VARCHAR(100),
    message_timestamp TIMESTAMP,
    message_type VARCHAR(20),          -- 'USER' | 'AI'
    message_content TEXT
);
